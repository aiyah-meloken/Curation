# Atlas — 标注指南（D 框架 / 读者人格）

> **核心命题**：标签的主语不是文章，是**读者**。
>
> 用户打开雷达看到的不是"今日新闻"，而是**这个 app 眼里的我**。每一个 L1 都在隐含地说："我认为你是一个会关心 X 的人。" 所以分类做不好，用户立刻觉得这是个普通新闻聚合；分类做对了，用户感到被看见。
>
> **L1 是领域无关的（domain-agnostic）。** 它描述的是读者的"阅读人格 / 阅读模式"，不假设用户在哪个行业。"巨头博弈"对 AI 从业者、律师、投资人都有意义；"风险预警"对工程师、安全从业者、企业主都有意义。打 tag 时不要悄悄假设用户是 AI 从业者。

---

## 一、给打 tag 的 Agent 的 Prompt

复制以下 prompt 给打 tag 的 LLM/Agent：

```
你正在为「Atlas 今日舆图」这个个人化 AI 资讯雷达打标签。

【最重要的规则】
标签描述的不是"这篇文章是什么"，而是"读这篇文章的我，是哪一面的我"。
- ❌ 错误（书店货架式）："这是一篇关于 AI 大模型的产业新闻"
- ✅ 正确（读者人格式）："读这篇文章的我，是个看大厂下棋的我"

【标签格式】
每张卡片打 1 个主标签，格式为 "L1/L2"：
- L1 = 读者人格（读这篇时我是谁）— 从下面 6 个固定值中选 1 个
- L2 = 主题领域（这一面的我今天在哪个领域看）— 从下面 14 个固定值中选 1 个

【设计哲学：MECE 主体身份轴】

L1 是 **mutually exclusive + collectively exhaustive** 的——按"事件主体类型"切，不混 cognitive role 轴。每张卡只能落一个 L1。

切法是按"事件主体的身份类型"：

```
事件主体是谁？
├─ 巨头公司动态（含 release / 战略 / 财务 / 政府关系 / 公司技术报告）
│      → 巨头博弈
├─ 中型公司 / 社区项目 / 全行业分析观察
│      → 行业脉搏
├─ Startup / 早期团队 / 个人项目 / 多 startup 赛道梳理
│      → 创业故事
├─ 学界论文 / 顶会 / 综述 / 研究社区
│      → 学术前沿
├─ 攻击者 / 威胁 / 漏洞
│      → 风险预警
└─ Lifestyle / 自然 / 真盲区
       → 盲区一瞥
```

【L1 = 6 张读者人格脸谱】

1. 巨头博弈 — 我盯大厂下棋
   何时选：**任何巨头公司的动态**——包括 model/SDK/产品 release、战略动作、财务、政府关系、公司技术报告、定价战略、合作合同、组织变动等
   信号词：投资 / 营收 / 战略 / 重组 / 反超 / 合作 / 致歉 / 诉讼 / 入华 / 联盟 / 双平台 / 发布 / 开源 / 限时 / 技术报告
   ⚠️ "巨头" 判定标准：模型万亿参数级、API 调用量国家级前列、被 OpenAI/Anthropic 视为对标对象、影响算力 stack 设计——满足 ≥2 项即按巨头处理。
   - 2026-04 的 AI 巨头：OpenAI、Anthropic、DeepSeek、Kimi（月之暗面）、字节、腾讯、阿里、华为、微软、苹果、Google、英伟达、英特尔、Meta、三星、小米
   - 反例：硅谷小独角兽（Cognition、Hyperliquid、Decagon、HeyGen、阶跃星辰、兔展、涂鸦、联影 等）即使估值高也不是巨头 → 行业脉搏
   - 跨界判断：汽车类（比亚迪、上汽大众、奥迪 等）的 EV 产品 release 不算 AI 巨头动态 → 行业脉搏/智驾出行
     - 但当文章主体是 AI 巨头切入跨界（华为乾崑切豪华燃油智驾、三星半导体切车规） → 巨头博弈/对应 L2

2. 行业脉搏 — 我读中型公司 / 社区 / 全行业的动态
   何时选：**所有非巨头、非 startup 的公司动态**——中型公司（HeyGen / 阶跃星辰 / 兔展 / 涂鸦 / 联影 / Momenta / 小马智行 等）的 release 与战略；**社区项目**（OpenClaw / Hugging Face / LangChain 类社区开源框架）的版本更新；**分析师/资本/记者长文**判断；趋势预测；行业全景观察；多家架构横向对比（如智算超节点 5 厂商对比）
   信号词：判断 / 趋势 / 观察 / 全景 / 调研报告 / 解读 / 反思 / 中型 / 社区 / 多家对比
   ⚠️ 与「巨头博弈」的区分：看主体是不是 AI 巨头（参见 1 中的判断标准）
   ⚠️ 与「创业故事」的区分：分析师写的纯思辨长文（无具体 startup 详细介绍，只有 thesis）→ 行业脉搏

3. 创业故事 — 我看 founder / 早期团队 / 个人项目在做什么
   何时选：startup 故事；早期团队动作（不是 OpenAI/Anthropic/字节这类巨头）；创业赛道分析；
        快速跟进开源（fast-follow）的小团队；创业人物路径；**赛道梳理（≥2 个 startup 详细介绍）**；个人开发者项目（如 Peter Steinberger 的 ClawSweeper）
   信号词：创业 / 团队 / 早期 / 公测 / 赛道 / fast-follow / founder / 个人项目
   ⚠️ **不要被"分析师视角"误导**：如果一篇文章详细介绍 ≥2 个 startup（含 founder/团队/融资/产品细节），
   即使带有编辑性的 thesis 框架（如"赛道趋势""标签半衰期"），仍属 **创业故事**。
   - 测试方法：删掉 startup 详细介绍后文章是否成立？不成立 → startup 是主体 → 创业故事
   - 反例：纯观点长文（无详细 startup 介绍，只有 thesis + 论据）→ 行业脉搏

4. 学术前沿 — 我盯研究苗头 / 学界论文 / 顶会 / 奖项 / 范式
   何时选：**学界发布的论文**（arXiv / ICLR / ACL / Nature / 顶会接收论文）；**综述论文**；顶会奖项 / 时间检验奖；研究社区聚集（讲堂 / Workshop）；范式信号事件
   ⚠️ 即使作者来自 Microsoft Research / Google Research / NVIDIA Research（巨头研究院），只要论文是纯研究方法、不指向公司战略产品 → **仍属学术前沿**
   - 反例：DeepSeek V4 自家技术报告（配合产品上线，是公司战略一部分）→ 巨头博弈/AI模型
   信号词：arXiv / ICLR / NeurIPS / ACL / Nature / 论文 / 综述 / 时间检验奖 / 杰出论文 / 讲堂

5. 风险预警 — 我得检查我的防御面
   何时选：CVE / 攻击事件 / 黑产 / 提示注入 / 模型滥用 / 凭证泄露 / Agent 安全攻击
   信号词：CVE / 漏洞 / 攻击 / 入侵 / 提示注入 / 木马 / 泄露 / 黑产
   ⚠️ L2 区分：
   - 攻击标的是模型/Agent（提示注入、Agent 劫持） → 模型安全
   - 攻击标的是传统系统（CVE、凭证、网络入侵），AI 只是工具 → 网络安全

6. 盲区一瞥 — 这条信息几乎没有任何专业读者会觉得对自己工作有直接影响
   何时选：**lifestyle / 消费奇闻 / 人文自然 / 跨界小事**——内容**不属于任何专业 vertical 的核心事件**，读者反应是"知道一下就好"，一两句话级别就够
   ⚠️ 严格判断：**不要把"邻接行业但有专业含量的内容"塞进盲区**。
   - 智驾新车型上市 → 行业脉搏/智驾出行（中型公司）或 巨头博弈/智驾出行（车厂巨头）
   - 折叠屏 + AI 助手范式信号 → 行业脉搏/AI助手聊天（专业含量，不是盲区）
   - 车规芯片巨头战略 → 巨头博弈/算力芯片（不是盲区）
   - **真盲区**：座头鲸恢复、高铁自行车随行、潮玩冷藏箱、消费可穿戴 lifestyle 评测——这些对几乎所有专业读者都是 perimeter
   信号词：野生动物 / 高铁 / 潮玩 / IP 经济 / 生活方式 / 偶发奇闻
   💡 长期：B 阶段后，"盲区"判断会让位给用户自己的兴趣模型——什么是真盲区由用户的阅读行为决定。

【已删除的旧 L1】

⚠️ 之前的「工具新货」「技术深挖」已被删除，原因是它们走的是 cognitive 轴而非 subject 轴，跟 巨头博弈/行业脉搏/创业故事 重叠，导致打 tag 不一致。
- 巨头公司的 release（V4 / GPT-5.5 / Hy3 等）→ 巨头博弈
- 中型公司/社区的 release（HeyGen / OpenClaw 等）→ 行业脉搏
- 个人项目（ClawSweeper）→ 创业故事
- 巨头公司技术报告（DeepSeek V4 技术报告及解读）→ 巨头博弈
- 多家架构对比（智算超节点）→ 行业脉搏

【L2 命名原则 — 重要】

L2 应该处于"具体性"与"可聚集性"的中间地带：

✅ 好的 L2 — 看到名字能猜到 70% 的内容
   - 「智驾出行」5 张：能猜到都是智驾/EV/出行硬件 → 准
   - 「网络安全」8 张：能猜到都是 CVE/攻击/黑产 → 准
   - 「创作工具」4 张：能猜到都是视频/图像/文档创作工具 → 准

❌ 太宽泛的 L2 — 名字猜不出实际内容
   - 「智能硬件」（已弃用）：理论上能塞智能音箱/机器人/家电/汽车，实际全是车 → 不准
   - 整改路径：换成符合实际内容的更窄名（智驾出行）

❌ 过度细分的 L2 — 一张卡一个独立桶
   - 把「大语言模型」硬拆成「DeepSeek 模型/GPT 模型/Kimi 模型」：每个都几张卡，且 L1 已经在做这个区分（巨头博弈聚合所有巨头公司的模型动态）→ 不需要

经验阈值：
- **每个 L2 至少 3 张卡才独立存在**；少于 3 张应该合并到语义相邻的 L2
- 例外：作为 fallback 的「其他」可以小（2 张以内），表明它是真正的 catchall
- 当 L1 已经在区分某个维度（公司/事件类型）时，**L2 不要再用同一维度切**——会和 L1 重复

允许的"正当 catchall" L2：
- 「AI 应用」（6 张）覆盖 3D/医疗/视频生成等单独成桶过细的垂直领域，本身是有意保留的伞型 L2
- 「AI 模型」（30+ 张）覆盖 LLM/多模态/语音的基础模型层，因为 L1 已在切公司/事件类型，L2 不再细分

---

【L2 = 14 个主题领域】（与 L1 正交，描述话题）

- AI模型          — 大模型本身（架构 / 训练 / 能力评测 / 开源生态 / 包括 LLM/多模态/语音基础模型）
                    💡 catchall 性质，因 L1 已切公司/事件维度，L2 不再细分
- AI Agent        — Agent 框架 / 应用 / 工程（含 Agent 行为研究、Agent 工具调用、记忆/约束机制）
- AI助手聊天      — 面向消费者的 AI 助手（ChatGPT / Claude / 豆包 / 涂鸦 Hey Tuya 类）
- 创作工具        — 视频 / 图像 / 文档创作类工具（GPT Image 2 / 兔展 UniWorld / Hyperframes / 剪映）
- AI应用          — 单独不够 3 张的垂直应用（医疗AI / 3D 重建 / 视频生成研究等）
                    💡 catchall 性质，避免 1-2 张的"医疗AI"独立 L2
- AI基础设施      — 数据中心 / 网络 / 训练系统（不含芯片本身）
- 算力芯片        — GPU / NPU / CPU / 车规芯片硬件本身
- 模型安全        — 提示注入 / Agent 劫持 / 模型对齐风险 / 模型被滥用做攻击
- 网络安全        — CVE / 黑产 / 凭证泄露 / 系统漏洞（标的不是 AI 本身）
- 智驾出行        — 智能驾驶 / 新能源车 / 出行硬件（智驾方案 / EV 新车型 / 域控制器 / 智驾 OEM 合作）
- 消费电子        — 手机 / 眼镜 / 潮玩 / 可穿戴（lifestyle 向消费产品）
- 商业策略        — 战略 / 组织 / 营收 / 投融资 / 市场调研（不指向具体技术）
- 政策监管        — 政府动作 / 法律 / 合规
- 其他            — 与上述都不强关联（座头鲸、高铁等真盲区类内容）

【打标流程 — 强制步骤】

**Step 0：必须读完整 content_md，不要只看 title + description。**
title 经常带有 marketing framing，description 是被截断的（~200 字）。基于这两者打 tag 的错误率约 17%——已被实证。在 tag-card-content.ts 里有完整 markdown，必读。

**Step 1：按 MECE 决策树定 L1**（见下面【MECE 决策树】节，按 6 步顺序判断）

**Step 2：选 L2 = 文章话题主导维度（不是技术机制）**
  - 一篇用 multi-agent 做视频剪辑的论文 → L2 是「AI 应用」（视频剪辑），不是「AI Agent」
  - 一篇用 LLM 做漏洞利用的研究 → L2 是「网络安全」（攻击标的），不是「AI Agent」
  - 一篇 Anthropic 内部项目泄露 → L2 看泄露的是什么（Conway = Agent 平台 → AI Agent）

**Step 3：风险预警的 L2 必须看攻击标的：**
  - 标的是模型/Agent（提示注入、越狱、Agent 劫持） → 模型安全
  - 标的是传统系统（CVE、凭证、网络入侵），AI 只是工具 → 网络安全

**Step 4：输出 `L1/L2`**

【MECE 决策树 — 重要】

每张卡按以下顺序判断，**先 hit 哪个就归哪个**：

**Step 1**：内容是不是 lifestyle / 偶发奇闻 / 真跨界（座头鲸、高铁、潮玩、消费可穿戴 lifestyle 评测等对所有专业读者都没专业含量）？
- 是 → **盲区一瞥**

**Step 2**：内容是不是 CVE / 攻击事件 / 黑产 / 提示注入 / 模型滥用？
- 是 → **风险预警**

**Step 3**：内容主体是不是学界论文（arXiv / ICLR / ACL / Nature / 顶会接收论文 / 综述）？
- 是 → **学术前沿**（即使作者来自巨头研究院）
- 例外：巨头公司自家发布的技术报告（不是 arXiv 论文，是产品配套）→ 巨头博弈

**Step 4**：内容主体是不是 startup / 个人项目 / ≥2 个 startup 详细赛道梳理？
- 是 → **创业故事**

**Step 5**：内容主体是不是巨头公司（OpenAI / Anthropic / DeepSeek / Kimi / 字节 / 腾讯 / 阿里 / 华为 / 微软 / 苹果 / Google / 英伟达 / 英特尔 / Meta / 三星 / 小米）的动态？
- 是 → **巨头博弈**
- 含 release / 战略 / 财务 / 政府关系 / 公司技术报告 / 定价战略 / 合作合同

**Step 6**：默认 → **行业脉搏**
- 中型公司动态 / 社区项目动态 / 多家架构对比 / 分析师长文

【关键陷阱】

- ⚠️ 巨头模型 release 跟分析师长文是不同 L1，但都涉及巨头：
  - 「OpenAI 发布 GPT-5.5」（公司动态） → 巨头博弈
  - 「DeepSeek 与 Kimi 互鉴史」（分析师对巨头的观察） → 行业脉搏
  - 区分：article 主体是公司动态本身，还是作者的观察思考？

- ⚠️ 跨界事件按主体识别：
  - 比亚迪/上汽大众 EV 新车型（汽车业事件，不是 AI 巨头动态） → 行业脉搏/智驾出行
  - 华为乾崑切豪华燃油智驾（华为是 AI 巨头切跨界） → 巨头博弈/智驾出行
  - 三星半导体切车规芯片（三星是巨头切跨界） → 巨头博弈/算力芯片

- ⚠️ 同一事件多张卡可以归不同 L1，按各自 framing 定：
  - DeepSeek V4 量子位实测 → 巨头博弈（DeepSeek 公司动态 + release）
  - DeepSeek V4 + 昇腾双平台 → 巨头博弈（同上 + 战略联盟）
  - 王兆洋解读 V4 → 巨头博弈（DeepSeek 公司技术报告解读）
  - 周一笑 DeepSeek×Kimi 撞车 → 行业脉搏（分析师对巨头的观察）

---

【冲突优先级 — 取最戏剧性的那张脸】

- 安全 CVE 文章里也讲了 Anthropic 政府合作 → 选 巨头博弈（Anthropic 政府订单 > CVE）
- DeepSeek V4 不同 framing 都是巨头博弈，但 L2 不同：
  - 「上线/限时优惠/API」 → 巨头博弈/AI 模型
  - 「算法/架构/KV cache」 → 巨头博弈/AI 模型（公司技术报告解读）
  - 「双平台/国产化/战略联盟」 → 巨头博弈/算力芯片
- Anthropic 内部项目泄露 → 巨头博弈/AI Agent（公司秘闻 > 风险预警，不是用户该 patch 的东西）
- 早期团队 fast-follow 大厂开源 → 创业故事（团队叙事 > release 本身）

【硬性禁止】

❌ 把学界论文（arXiv/ICLR/ACL）当巨头博弈或行业脉搏 → 应是 学术前沿（即使作者来自巨头研究院）
❌ 把投资交易/营收数据当 AI 模型 L2 → 应是 商业策略 L2
❌ 把邻接行业（智驾/折叠屏/车规芯片）塞进盲区一瞥 → 应按主体识别（车厂/AI 巨头跨界）
❌ 把汽车/EV 新车型按 AI 巨头标准识别 → EV 厂商是 EV 行业巨头，但在 AI 上下文中是邻接行业 → 行业脉搏/智驾出行
❌ 同一篇打多个 L1（取一个最戏剧性的）
❌ 发明新 L1（找不到时用 盲区一瞥/其他）

【极重要】

- L1 = 读者人格（domain-agnostic，不假设 AI 从业者）
- L2 = 话题领域（与 L1 正交）
- 输出格式严格：`L1/L2` 单字符串，无空格
- L1 名字本身是产品力，不要改名（"创业故事" 不是 "商业新闻"，"巨头博弈" 不是 "大公司动态"）。
```

## 二、为什么是这套（设计理由）

### 为什么 L1 是「读者人格」而不是「文章属性」

之前的 4 个 perspective（领域 / 内容 / 技术层 / 信号）都是**书店货架**：站在文章的角度描述属性。这导致：

- 用户看到 "AI 大模型 43 张" 时，**没有情感反应** —— 它只是说"今天有 43 篇关于这个领域的文章"
- 而看到 "巨头博弈 18 张" 时，会有反应 —— "原来我今天看了这么多大厂下棋"

第二种感受才是 **app 懂我** 的来源。L1 在告诉用户：**"我看到了你的这一面"**。

### 为什么是这 8 个

来自 105 张卡的实际分布演化，不是先验设计。最初尝试 8 张脸（含 工具新货 / 技术深挖），发现这两张走 cognitive 轴跟其他主体身份脸重叠，于是删除 → 6 张纯主体身份脸。具体演化见第三点四节。

### 为什么 L1/L2 正交

L1 = 读者**视角**（按主体身份切），L2 = **话题**（与主体正交）。同一个话题（AI 大模型）可以分布在多个 L1：
- DeepSeek V4 算法分析 → `巨头博弈/AI模型`（DeepSeek 是巨头，公司技术报告）
- 兔展 UniWorld V2.5 发布 → `行业脉搏/创作工具`（兔展是中型公司）
- LLM DNA ICLR 论文 → `学术前沿/AI模型`（学界论文）
- DeepSeek 与 Kimi 互鉴史 → `行业脉搏/AI模型`（分析师对巨头的观察）

这种正交性是 D 框架的核心 —— 用户既能问 "今天我哪些脸被喂养了"（L1 视角），也能问 "AI 模型这个话题今天我从哪几个视角接触了"（L2 话题）。

### 为什么没有「AI/XX赛道」这种三层

最初考虑过 `创业故事/AI/Anthropic 融资` 三层结构。但发现：
- L3（具体赛道/事件主体）天然是文章里的实体，不是分类
- 强制三层会稀释 L2，每个 L1 下出现太多 L2，地图渲染塌陷
- 实际打 tag 时，L2 已经足够回答"在哪个领域"

L3 留给搜索/筛选维度（直接在标题/描述里搜实体），不进 taxonomy。

## 三点四、L1 演化历史（参考）

A 阶段经历过两次重要重构，记录留供参考：

### V1（旧）：8 张 L1，混 subject + cognitive 轴

8 个 L1：技术深挖 / 工具新货 / 巨头博弈 / 行业脉搏 / 学术前沿 / 风险预警 / 创业故事 / 盲区一瞥

**问题**：技术深挖 + 工具新货 走的是"读者认知动作"轴（"我读架构" / "我可以用"），跟 巨头博弈/行业脉搏/创业故事 的 subject 轴重叠：
- 「DeepSeek V4 发布」既是 OpenAI 巨头动态（巨头博弈），又是"读者可以用"（工具新货）
- 「DeepSeek V4 技术报告」既是 DeepSeek 巨头动态（巨头博弈），又是"读者可以读架构"（技术深挖）

打 tag 不一致。04-25 vs 04-26 工具新货占比从 7% 跳到 38%（+31pp）——boundary 不锐利。

### V2（当前）：6 张 L1，纯 MECE 主体身份轴

删除 工具新货 + 技术深挖，按主体大小重新分配 27 张卡：
- 巨头公司 release / 公司技术报告 → 巨头博弈
- 中型公司 + 社区项目 / 多家架构对比 → 行业脉搏
- 个人项目 → 创业故事

漂移修复后：04-25 vs 04-26 巨头博弈 33% / 38% (+5.5pp)，行业脉搏 24% / 28% (+3.5pp)——剩余漂移反映真实新闻分布。

## 三点五、打 tag 历史经验（仍有效的判别准则）

**1. 必须读完整 content_md** — 基于 title + 摘要的错误率约 17%（被实证）。

**2. 风险预警的 L2 看攻击标的：**
- 标的是模型/Agent → 模型安全
- 标的是传统系统 → 网络安全
（例：Claude Opus 写 Chrome 漏洞链 → 标的是 Chrome → 风险预警/网络安全）

**3. L2 选话题主导维度，不是技术机制：**
- 一篇用 multi-agent 做视频剪辑的论文 → L2 是 AI 应用（视频剪辑），不是 AI Agent
- 一篇 Anthropic Conway 泄露 → L2 看泄露的是什么（Conway = Agent 平台 → AI Agent）

**4. ≥2 个 startup 详细介绍的赛道梳理 → 创业故事**：
- 即使带"标签半衰期" thesis 框架，仍属创业故事
- 测试：删掉 startup 详细介绍后文章是否成立？不成立 → startup 是主体 → 创业故事

**5. 跨界硬件按主体识别：**
- 智驾/EV 新车型（车厂为主体） → 行业脉搏/智驾出行
- 华为乾崑切豪华燃油智驾（华为 AI 巨头切跨界） → 巨头博弈/智驾出行
- 三星半导体切车规芯片（三星巨头切跨界） → 巨头博弈/算力芯片

## 四、L1 演化条件（什么时候该改这个表）

A 阶段固定 6 个 L1。下面任一信号出现就需要重审：

1. **某 L1 连续 3 天 ≤ 2 张** → 该 L1 可能不该独立存在（合并 / 删除）
   - 当前候选：创业故事、盲区一瞥
2. **某 L1 连续 3 天 > 40%** → 该 L1 太粗，需要拆分
   - 当前候选：巨头博弈（35%，临界值附近——如果持续高位且 L2 集中度低，考虑按公司/事件类型拆分）
3. **同一个 L1 内 L2 数量 > 8** → L1 需要拆分
4. **新出现的卡反复落进「盲区一瞥/其他」≥ 5 张** → 需要新增 L1

进 B 阶段（个性化）后，这些信号由用户自己的行为决定，不再是全局阈值。

## 五、与之前 P1-P4 的关系

之前的 P1（领域）/ P2（内容）/ P3（技术层）/ P4（信号）**已弃用**。它们都是书店货架式分类，无法承担"懂我"的产品体验。

P2/P3/P4 的部分语义被吸收进当前 L1：
- 「论文精读 / 架构分析」→ 学术前沿（学界）/ 巨头博弈（公司技术报告）
- 「产品发布 / 新版本发布」→ 工具新货
- 「商业交易」→ 巨头博弈
- 「安全漏洞」→ 风险预警

P3（技术层）已被 L2（主题领域）吸收。
P4（信号类型）已被 L1（读者人格）吸收 —— "新攻击手法出现" 不再是描述事件，而是触发"风险预警"这张脸。
